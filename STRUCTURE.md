# DD Learn — サイト構成定義

このファイルを編集すると、サイト構成を変更できます。  
編集後、AI に「STRUCTURE.md に従って index.html を更新して」と依頼してください。

セクションの **順序・有無・内容** を自由に変えてOKです。

---

## ファイル構成

```
dd-learn/
├── index.html
├── config.json
├── apps.json
├── manifest.json
├── sw.js
├── icon-192.png / icon-512.png / icon.svg
└── assets/
    ├── hero-banner-illustrated.png  （子供イラスト入りバナー）
    ├── hero-banner-simple.png       （シンプル版バナー）

```

## グローバル要素（全ページ共通）

### HEADER（画面上部に固定）

- **位置**: 画面上部、スクロールしても固定（sticky）
- **左側**: ロゴ画像 (`assets/logo.png`)
- **右側**: 
  - ステータスバッジ（緑ドット + appTagline）※モバイルでは非表示
  - 未ログイン時: Google公式ログインボタン
  - ログイン済み: ユーザーチップ（アバター + 名前）→ タップでログアウトボタン展開

### FOOTER（画面下部）

- 中央寄せのコピーライト表記
- `config.json` の `footerText` から取得

## メインコンテンツ（上から順）

### 1. HERO

- **目的**: アプリのキャッチビジュアル（最上部）
- **2モード**:
  - **画像モード（デフォルト）**: `config.json` の `heroBanner` に画像パスが指定されている場合、その画像を全幅表示
  - **テキストモード**: `heroBanner` を空 `""` にすると、heroEyebrow / heroTitle / heroDesc を表示する従来形式
- **使える画像**:
  - `assets/hero-banner-illustrated.png`（子供イラスト入り）
  - `assets/hero-banner-simple.png`（シンプル版）

### 2. LOGIN_TEASER ⭐ 未ログイン時のみ

- **位置**: HEROの直下
- **目的**: ログインを促すバナー
- **デザイン**: 黄色系、点線ボーダー、🔒アイコン

### 3. DASHBOARD ⭐ ログイン時のみ

- **位置**: HEROの直下（LOGIN_TEASERと入れ替わり表示）
- **構成**:
  - **ヘッダー**: 「📊 きょうのせいか」（左） + 「⭐ N pt（教科名）」バッジ（右）
  - **教科フィルターピル**: 横スクロール
    - `ぜんぶ`（青、デフォルト選択）
    - `🧮 さんすう`、`📖 こくご`、`🔬 りか`、`🌏 しゃかい`、`🌟 えいご` ...（apps.jsonの `type: "subject"` から自動生成）
    - 選択時は教科色で塗られて立体的になる
  - **折れ線グラフ**: 過去 N 日（デフォルト7日）。選択中の教科に応じて色とデータが変わる
- **データ取得**: localStorage から教科別ポイントを読み込み
- **タップで切替**: ピルをタップすると、今日のポイント数とグラフが瞬時に切り替わる

### 4. APP_GROUPS（教科グループ × N）

- **データソース**: `apps.json`
- **構成**: 各グループは「グループバッジ」+「カード行」
- **カード**:
  - 上部: 教科色のアクセントバー
  - アイコン + 「出発」ボタン
  - アプリ名 / 説明 / タグ（レベル + 報酬）
- **タップ動作**: ビューア（フルスクリーン iframe）でアプリを開く
- **科目外グループ** (`type: "general"`): 紫系の配色で視覚的に区別

## オーバーレイ要素（必要時に表示）

### VIEWER（アプリ表示）

- **トリガー**: カードをタップ
- **位置**: 全画面オーバーレイ（z-index: 1000）
- **構成**:
  - 上部バー（高さ60px）: アイコン+アプリ名 / 「⭐ 完了！」（ログイン時のみ） + 「閉じる」
  - 本体: iframe

### TOAST（ポイント獲得通知）

- **トリガー**: 「⭐ 完了！」ボタン押下
- **テキスト例**: 「⭐ +20 ポイント！ きょうの合計: 35pt」
- **表示時間**: 約2.8秒で自動消去
- **特殊**: 報酬 0pt のアプリ（便利ツール）の場合は「お疲れさま！」と表示

## ポイント記録の仕様

### データ構造（localStorage）

```
キー: mq_{googleUID}_points
値:   {
        "2026-04-25": { "math": 30, "japanese": 20 },
        "2026-04-24": { "math": 50 }
      }
```

- 日ごと × 教科ID別の二重マップ
- 教科IDは apps.json の各グループの `id`（"math" / "japanese" / "science" / "social" / "english" など）
- 科目外（general）アプリのポイントは `_other` キーに集約
- 旧フォーマット（`{date: 数値}`）は `{date: {_legacy: 数値}}` に自動変換

### 完了時の挙動

1. ユーザーが「⭐ 完了！」をタップ
2. `apps.json` の `reward` から数値を抽出（例: "⭐ 20" → 20pt）
3. localStorage の該当日・該当教科に加算
4. グラフと「今日のポイント」表示を更新
5. トースト表示

## 表示状態の切り替えルール

| 要素 | 未ログイン | ログイン済み |
|---|---|---|
| Googleログインボタン | 表示 | 非表示 |
| ユーザーチップ | 非表示 | 表示 |
| LOGIN_TEASER | **表示** | **非表示** |
| DASHBOARD（フィルター含む） | **非表示** | **表示** |
| カード | タップで開く | タップで開く |
| 「⭐ 完了！」ボタン | 非表示 | 表示 |
| ポイント記録 | 行わない | 端末localStorageに保存 |

## 配色テーマ

| 用途 | 色 |
|---|---|
| プライマリ（テーマ色） | `#39a7ff` |
| アクセント | `#ff8a00` |
| 強調（星・報酬） | `#ffd34d` |
| インクテキスト | `#26324d` |
| 補足テキスト | `#667085` |

教科ごとの色は `apps.json` の各グループの `color` フィールドで定義。フィルターピルとグラフの色も連動する。

---

## アセット管理ルール

新しい画像を追加するときは:
1. `assets/` フォルダに保存
2. ファイル名は用途がわかる英数小文字（例: `hero-banner-spring.png`）
3. 大きすぎる画像はリサイズ（横幅 1600px 以下推奨）
4. PWA でオフライン使用するなら `sw.js` の `STATIC_ASSETS` 配列に追記
5. config 系で参照するならパスを `config.json` に記載

## 編集例

**例1: ヒーローバナーをシンプル版に変える**
→ `config.json` の `heroBanner` を `"assets/hero-banner-simple.png"` に書き換え

**例2: 折れ線グラフの期間を 14 日に増やす**
→ `config.json` の `pointsChartDays` を `14` に変更

**例3: 「今週のがんばりランキング」セクションを追加したい**
→ 新規セクションを追加：
```
### N. WEEKLY_RANKING ⭐ ログイン時のみ
- **目的**: 今週よくがんばった教科を表示
- **構成**: 教科アイコン + 獲得ポイント数の棒グラフ
- **データソース**: localStorage の過去7日分から教科別合計を計算
```
→ AI に「この変更を実装して」と依頼
