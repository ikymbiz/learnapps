# DD Learn — サイト構成定義

このファイルを編集すると、サイト構成を変更できます。  
編集後、AI に「STRUCTURE.md に従って index.html を更新して」と依頼してください。

セクションの **順序・有無・内容** を自由に変えてOKです。

---

## ファイル構成

```
dd-learn/
├── index.html
├── config.json
├── apps.json
├── manifest.json
├── sw.js
├── icon-192.png / icon-512.png / icon.svg
└── assets/                        ← 画像素材
    ├── hero-banner-illustrated.png  （子供イラスト入り）
    └── hero-banner-simple.png       （シンプル版）
```

## グローバル要素（全ページ共通）

### HEADER（画面上部に固定）

- **位置**: 画面上部、スクロールしても固定（sticky）
- **左側**: アプリロゴ（⭐ + appName + appSubtitle）
- **右側**: 
  - ステータスバッジ（緑ドット + appTagline）※モバイルでは非表示
  - 未ログイン時: Google公式ログインボタン
  - ログイン済み: ユーザーチップ（アバター + 名前）→ タップでログアウトボタン展開

### FOOTER（画面下部）

- 中央寄せのコピーライト表記
- `config.json` の `footerText` から取得

## メインコンテンツ（上から順）

### 1. HERO

- **目的**: アプリのキャッチビジュアル（最上部）
- **2モード**:
  - **画像モード（デフォルト）**: `config.json` の `heroBanner` に画像パスが指定されている場合、その画像を全幅表示。テキスト要素は非表示。
  - **テキストモード**: `heroBanner` を空 `""` にすると、heroEyebrow / heroTitle / heroDesc を表示する従来形式に戻る。
- **使える画像** (assets/ フォルダ):
  - `assets/hero-banner-illustrated.png` （子供イラスト入りバナー）
  - `assets/hero-banner-simple.png` （シンプル版）
- **画像差し替え**: `config.json` の `heroBanner` 値を書き換えるだけで切替可能。

### 2. LOGIN_TEASER ⭐ 未ログイン時のみ

- **位置**: HEROの直下
- **目的**: ログインを促すバナー
- **デザイン**: 黄色系、点線ボーダー、🔒アイコン
- **テキスト**: 
  - タイトル: 「ログインすると成果が見えるよ！」
  - 補足: 「きょうのポイントと、1週間のがんばりグラフが表示されます。」

### 3. DASHBOARD ⭐ ログイン時のみ

- **位置**: HEROの直下（LOGIN_TEASERと入れ替わり表示）
- **目的**: 今日のポイントと過去N日（デフォルト7日）の折れ線グラフ
- **構成**:
  - ヘッダー: 「📊 きょうのせいか」（左） + 「⭐ 今日のポイント数」バッジ（右）
  - グラフ本体: Chart.js による折れ線グラフ（高さ160px）
  - 最新の点（今日）はオレンジ色で大きめに強調

### 4. APP_GROUPS（教科グループ × N）

- **目的**: 教科ごとの学習アプリ一覧
- **データソース**: `apps.json`
- **構成**: 各グループは以下の形：
  - **グループバッジ**: アイコン + 教科名（色枠付き、立体感のある影）
  - **カード行**: 横スクロール（モバイル）/ 折り返しグリッド（デスクトップ）
- **カード**:
  - 上部: 教科色のアクセントバー
  - アイコン（左上） + 「出発」ボタン（右上）
  - アプリ名（太字）
  - 説明文
  - タグ（レベル + 報酬）
- **タップ動作**: ビューア（フルスクリーン iframe）でアプリを開く
- **科目外グループ** (`type: "general"`): 紫系の配色で視覚的に区別

## オーバーレイ要素（必要時に表示）

### VIEWER（アプリ表示）

- **トリガー**: カードをタップ
- **位置**: 全画面オーバーレイ（z-index: 1000）
- **構成**:
  - 上部バー（高さ60px）:
    - 左: アイコン + アプリ名
    - 右: 「⭐ 完了！」ボタン（ログイン時のみ） + 「閉じる」ボタン
  - 本体: 該当URLを表示する iframe

### TOAST（ポイント獲得通知）

- **トリガー**: 「⭐ 完了！」ボタン押下
- **位置**: 画面下部中央、ボタンより少し上
- **テキスト例**: 「⭐ +20 ポイント！ きょうの合計: 35pt」
- **表示時間**: 約2.8秒で自動消去

## 表示状態の切り替えルール

| 要素 | 未ログイン | ログイン済み |
|---|---|---|
| Googleログインボタン | 表示 | 非表示 |
| ユーザーチップ | 非表示 | 表示 |
| LOGIN_TEASER | **表示** | **非表示** |
| DASHBOARD | **非表示** | **表示** |
| カード（一般） | タップで開く | タップで開く |
| 「⭐ 完了！」ボタン | 非表示 | 表示 |
| ポイント記録 | 行わない | 端末localStorageに保存 |

## 配色テーマ

| 用途 | 色 | 備考 |

| プライマリ（テーマ色） | `#39a7ff` | 空色 |

| 強調（星・報酬） | `#ffd34d` | イエロー |

| 補足テキスト | `#667085` | グレー |

## レスポンシブ仕様

- **760px 以下（モバイル）**:
  - HERO: 画像は全幅で自動スケール、テキスト版は1カラム表示
  - ヘッダーバッジ非表示
  - カード行: 横スクロール（スナップ付き）

- **700px 以上（デスクトップ）**:
  - カード行: 折り返しグリッド（最小幅230px）

---

## アセット管理ルール

新しい画像を追加するときは:
1. `assets/` フォルダに保存
2. ファイル名は用途がわかる英数小文字（例: `hero-banner-spring.png`, `bg-pattern-stars.svg`）
3. 大きすぎる画像はリサイズ（横幅 1600px 以下推奨）
4. PWA でオフライン使用するなら `sw.js` の `STATIC_ASSETS` 配列に追記
5. config 系で参照するならパスを `config.json` に記載

## 編集例

**例1: ヒーローバナーをシンプル版に変える**
→ `config.json` の `heroBanner` を `"assets/hero-banner-simple.png"` に書き換え

**例2: ヒーローを画像ではなくテキストに戻す**
→ `config.json` の `heroBanner` を `""`（空文字）にする

**例3: 「今週のがんばりランキング」セクションを追加したい**
→ 新規セクションを追加：
```
### N. WEEKLY_RANKING ⭐ ログイン時のみ
- **目的**: 今週よくがんばった教科を表示
- **構成**: 教科アイコン + 獲得ポイント数の棒グラフ
- **データソース**: localStorage の過去7日分から教科別合計を計算
```
→ AI に「この変更を実装して」と依頼すれば追加されます。
